Payment Flow

End-to-End Process

01

Connect Wallet

Connect any supported Solana wallet: Phantom, Solflare, Binance Web3, OKX, Trust Wallet, Bybit, or Gate.io. SOLQ never touches private keys.

02

Scan QRIS Code

Point camera at any standard QRIS code — GoPay, Dana, BCA, or any merchant. SOLQ parses the EMVCo payload and detects Dynamic vs Static QRIS automatically.

03

Review Live Quote

CoinGecko oracle provides real-time pricing. Jupiter quotes are verified against market rates with less than 2% tolerance. Slippage fixed at 0.5%.

04

Authorize & Sign

Tap "Launch External Wallet". Your wallet app signs the transaction. SOLQ only requests authorization — it never executes without your explicit signature.

05

On-Chain Execution

Jupiter ExactOut swap executes: SOL/USDC → IDRX. Settlement is delegated to licensed partner rails. Blockchain confirmation in approximately 3–10 seconds.

06

Settlement Complete

The backend polls Solana RPC to confirm transaction finality. The "Settlement Completed" screen appears only after on-chain confirmation is secured.

Core Implementation

Technical Architecture

Universal Wallet Connectivity

Supports Phantom, Solflare, Binance Web3, OKX, Trust Wallet, Bybit, and Gate.io via Android Intent Filters. Protocol uses deep linking — SOLQ never has access to private keys.

Real-Time Oracle Pricing

CoinGecko API powers live price feeds. Jupiter quotes are cross-validated against market rates with a strict tolerance of less than 2%. Slippage is fixed at 0.5% for reliability.

On-Chain Settlement Abstraction

Flow: SOL/USDC → Jupiter ExactOut Swap → IDRX (Rupiah Stablecoin) → Licensed Settlement Partner. Backend confirms finality via Solana RPC polling before completing the transaction.

QRIS Intelligence — Mata Pinter

EMVCo-compliant QRIS parser. Detects Tag 54 for Dynamic QRIS (amount locked) vs. Static QRIS (user inputs amount). Routes settlement via extracted merchant PAN and account identifiers.

Auditable State Machine

All payments follow a strict, auditable state sequence: CREATED → AUTHORIZATION_REQUESTED → AUTHORIZED → AWAITING_SETTLEMENT → COMPLETED. No state skipping. No ambiguous transitions.

Non-Custodial by Design

SOLQ does not store balances, does not custody funds, does not act as an e-wallet, and does not issue QR codes. It only requests authorization, orchestrates execution, and delegates settlement.
